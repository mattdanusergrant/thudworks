# Session 2026-06-20 — Builder

#LLM-generated

**Task:** Spike "Formula mode" — a tixy-style box where typed math draws the beat
(morphing ThudWorks toward "Math Draws Graphics", but for rhythm).

**Status:** done.

**What shipped:**
- New `ƒ(x,y,t)` input under the presets bar. Per cell, evaluates the expression with
  `x`=step (0–15), `y`=row (0–8, 0=top/Kick), `i`=`y*16+x`, `t`=bar counter. Cell fires
  when the result is `> 0`. `Math` is in scope via `with (Math)` inside a `new Function`
  body (sloppy-mode, like tixy), so `sin`/`floor`/`PI` work bare.
- Live: grid redraws on every keystroke; invalid expressions flag the box red (no crash).
- Generative: while playing, `t` increments each bar, so formulas using `t` evolve. The
  per-bar recompute hooks `triggerColumn` at `step === 0`.
- Disengages on manual cell edit / Clear / Random / Preset (you take control back).
- Five starter-formula chips.

**Files:** `app.js` (formula engine + wiring), `index.html` (section), `style.css`
(panel styling), rebuilt `thudworks-standalone.html`, README "Formula mode" section.

**Decisions made:**
- Threshold is `value > 0` (matches tixy's "positive = on").
- Per-bar recompute also redraws via `syncGrid` at schedule time, so the visual leads
  audio by ~one 16th. Acceptable for a spike; revisit if it ever feels off.
- `new Function` on user input is fine here — client-side only, their own browser (tixy
  does the same).

**Blockers:** none. Not manually run in a browser from this environment — verified by
`node --check` on the merged bundle and grep that the formula code inlined. Worth a quick
click-test on the live site once Pages is enabled.
