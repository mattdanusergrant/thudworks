#LLM-generated

**Task:** Spike "Formula mode" — a tixy-style box where typed math draws the beat
(morphing ThudWorks toward "Math Draws Graphics", but for rhythm).

**Status:** done.

**What shipped:**
- New `ƒ(x,y,t)` input under the presets bar. Per cell, evaluates the expression with
  `x`=step (0–15), `y`=row (0–8, 0=top/Kick), `i`=`y*16+x`, `t`=bar counter. Cell fires
  when the result is `> 0`. `Math` is in scope via `with (Math)` inside a `new Function`
  body (sloppy-mode, like tixy), so `sin`/`floor`/`PI` work bare.
- Live: grid redraws on every keystroke; invalid expressions flag the box red (no crash).
- Generative: while playing, `t` increments each bar, so formulas using `t` evolve. The
  per-bar recompute hooks `triggerColumn` at `step === 0`.
- Disengages on manual cell edit / Clear / Random / Preset (you take control back).
- Five starter-formula chips.

**Files:** `app.js` (formula engine + wiring), `index.html` (section), `style.css`
(panel styling), rebuilt `thudworks-standalone.html`, README "Formula mode" section.

**Decisions made:**
- Threshold is `value > 0` (matches tixy's "positive = on").
- Per-bar recompute also redraws via `syncGrid` at schedule time, so the visual leads
  audio by ~one 16th. Acceptable for a spike; revisit if it ever feels off.
- `new Function` on user input is fine here — client-side only, their own browser (tixy
  does the same).

**Blockers:** none. Not manually run in a browser from this environment — verified by
`node --check` on the merged bundle and grep that the formula code inlined. Worth a quick
click-test on the live site once Pages is enabled.

## Major pivot (same day) — code-to-song environment

**Task:** abandon the grid; make ThudWorks a pure code screen that generates whole songs,
with example songs ready to click.

**Status:** done.

**What it is now:** a text editor where you write a song in a tiny JS-based DSL, press
Play, and it compiles + loops. Example songs load and play on click.

**The language** (`song.js`):
- `tempo(bpm)`, `swing(0–70)`, and `play(instrument, pattern, opts?)` per part.
- Patterns are one 16th-note per cell. Drums: char-per-step (`x` hit / `.` rest, spaces
  ignored). Pitched: whitespace tokens — note (`C4`/`F#3`/`Bb2`), `.` rest, `-` tie (sustain).
- Every part loops to fill the longest part, so short loops tile under long melodies.
- Helpers `rep(p,n)` and `euclid(hits,steps,rot)`. opts `{wave,cutoff,detune,gain}`.
- Instruments: kick/snare/clap/hat/openhat/cowbell/clave/tom + pitched bass/synth/lead/pad/pluck.
- Compiled via `new Function(...)` with the helpers in scope — client-side, their browser
  (same trust model as the earlier formula mode).

**Engine:** `SongPlayer` flattens all tracks to one sorted event list (notes get real
durations from tie counting), then a 25ms lookahead scheduler fires them and loops the song.

**Synth:** added a melodic `synth` voice to `synth.js` (two detuned oscillators → lowpass
→ AD envelope; `dur` set per note from ties). `lead`/`pad`/`pluck` are presets over it.

**Examples** (`examples.js`): Boom Bap, Acid House, Trap, Lo-Fi, Euclid Techno — full
arrangements, click to load + play.

**Files:** rewrote `app.js`, `index.html`, `style.css`; new `song.js`, `examples.js`;
extended `synth.js`; removed `sequencer.js`; updated `build-standalone.py` + README.

**Decisions made:**
- Grid/formula UI fully removed (the pivot superseded it). The formula-mode work earlier
  today is gone from the live app but stays in git history.
- Parts loop to the longest part (tracker feel, minimal typing) rather than all-equal-length.

**Verification:** rebuilt the standalone bundle; `node --check` clean; ran a headless test
that compiles all 5 examples and flattens them to events (event counts/bar counts sane,
times monotonic from 0, error paths throw). **Not yet clicked in a real browser** — worth
an audio listen once Pages is on.

**Blockers:** GitHub Pages still needs enabling by the Operator (Settings → Pages → main / root).

## Arrangement + per-part swing + URL share (same day)

**Task:** add master song length, section markers, swing-per-part, and shareable links.

**Status:** done.

- **`length(bars)`** — fixes total song length; player uses `song.cells || longest part`.
- **`seq(...parts)`** — glues pattern strings in order (sections). With `rep` and
  `rep(".", n)` for rests, you arrange intro/drop/etc.
- **Per-part swing** — `play(inst, pat, { swing: 0–70 })` overrides the global `swing()`
  for that part only (clamped in the player).
- **Share** — a button base64-encodes (UTF-8-safe) the editor code into `#s=…`, updates
  the URL, and copies the link; on boot a `#s=` hash loads into the editor before the
  default example. No server, no storage.
- New **Arranged** example shows `length` + `seq` + per-part swing + chords together.
  Cheat sheet + README updated. `.ghost` button style re-added for Share.

**Verification:** rebuilt standalone, `node --check` clean. Headless test: all 7 examples
compile; `length(8)`→8 bars; per-part swing delays cell 1 to 0.1875s as expected; `seq`
of two 1-bar parts → 2 bars; base64 UTF-8 share round-trip is lossless. Still not yet
clicked/heard in a real browser (Pages not enabled).

## Section arrangement + WAV download (same day)

**Task:** more arrangement power; make every example a fully arranged song; replace Share
with "download as MP3".

**Status:** done — shipped as **WAV** (Operator chose dependency-free over bundling an MP3
encoder; browsers have no native MP3 encoder, and WAV keeps the no-deps/no-files identity).

**Arrangement primitives (`song.js`):**
- `section(name, { instrument: pattern | [pattern, opts] })` — define a reusable block; its
  length = the longest part in it.
- `arrange(...names)` — sequence blocks into the whole song; each part is tiled at the
  cell level to exactly fill its block, so all tracks stay bar-aligned.
- `transpose(pattern, semitones)` — shift a pitched pattern (chords + ties pass through).

**Examples:** all 7 rewritten as full arrangements (intro / groove / variation / etc.) via
section+arrange; the Arranged one also shows per-part swing + chords + a transposed chorus.

**Download (`app.js`):** removed Share/URL code. New "⬇ Download WAV" renders the song with
`OfflineAudioContext` (rebuilds the kit + compressor offline, schedules every event at its
absolute time, +0.6s tail) and writes a 16-bit PCM WAV via a hand-rolled `bufferToWav`
(no libraries). Status shows "rendering…"/"downloaded ✓"; a `rendering` flag pauses the
play-status updater.

**Decisions:** WAV not MP3 (no-deps); arrange tiles at cell level (not string rep) to
guarantee alignment.

**Verification:** `node --check` clean; headless tests prove all 7 examples are bar-aligned
(every track equal length, multiple of 16), `transpose('A1 C2+E2 ...', 5)` === the hand-written
`D2 F2+A2 ...`, and `bufferToWav` emits a valid RIFF/WAVE header (2ch/44100/16-bit, exact byte
lengths). The OfflineAudioContext render + actual audio still need a real browser (Pages off).
